## 1. 核心定位与设计哲学 (Core Philosophy)

| 维度 | 决策 (Decision) | 理由 (Rationale) |
|------|-----------------|------------------|
| **定位** | 统一操作系统（Unified OS） | 不仅是工具拼凑，而是覆盖所有工作场景的底层平台。 |
| **价值主张** | 一个平台 = 办公 + 开发 + 运维 + 知识 | 整合碎片化需求，实现无缝衔接。 |
| **成本** | 永远免费 | 降低用户门槛，消除商业顾虑，专注技术价值。 |
| **运行** | 永远本地运行（Local-First） | 数据主权、隐私保护、离线可用。没有云依赖。 |
| **控制** | IM 远程控制 (Feishu/WeChat/DingTalk) | 随时随地通过指令控制本地复杂操作。 |
| **Agent** | 8+ 专用 Agent | 通过状态机驱动，协同完成复杂任务流。 |
| **技术栈** | Node.js + TS + SQLite + Ollama | 轻量、高性能、开源、无需云端依赖。 |
| **交付** | 12 周分阶段交付 | 敏捷迭代，快速上线核心闭环。 |

## 3. 核心功能细节 (Functional Specs)

### 3.1 Agent 控制台 (Agent Console)
*   **功能**: 状态监控（运行/idle/error）、CPU/内存资源占用、任务队列。
*   **操作**: 启动 (Start)、停止 (Stop)、重启 (Restart)、配置 (Config)、日志查看 (Logs)。
*   **状态机**: Idle -> Pending -> Running -> Completed。

### 3.2 任务与流程 (Task & Workflows)
*   **看板**: 支持拖拽的任务看板系统（待处理/进行中/已完成）。
*   **工作流**: 自定义自动化流程，跨部门/团队/项目无缝协同。

### 3.3 服务器管理 (Server Monitor)
*   **实时状态**: CPU、RAM、Disk、网络 I/O 实时刷新。
*   **操作**: 部署 (Deploy, Docker/K8s)、诊断 (Diagnose)、端口/网络管理。

### 3.4 统一文件与知识库 (Knowledge & Files)
*   **文件管理**: VS Code 级的文件树浏览、搜索、右键菜单（支持本地及 NAS/SMB 挂载）。
*   **知识库**: 基于 Markdown/Rich Text 的文档管理、标签、分类。支持 AI 生成（本地 Ollama）。

### 3.5 IM 集成 (IM Remote Control)
*   **平台支持**: 飞书 (Lark), 企业微信 (WeCom), 钉钉 (DingTalk)。
*   **指令控制**: 通过 Webhook 解析 IM 指令（如 `/status`, `/deploy`），实现跨端远程操控。
