# Worker Agent 平台 · 全景功能设计书 v2.0

> **一句话定位**：一个平台，覆盖办公×开发×运维×知识——像操作系统一样简单，像工作流引擎一样强大。
>
> **设计哲学**：默认展示最小信息，需要时展开。所有复杂都藏在简单之后。

---

## 一、平台愿景

### 1.1 为什么需要这个平台

现在的开发者和办公用户每天在 **10+ 个工具之间反复切换**：

```
开发：代码编辑器 → Git客户端 → 终端 → Docker桌面 → 部署平台
办公：即时通讯 → 日历 → 文档 → 邮件 → 文件存储
运维：监控面板 → 服务器终端 → 日志平台 → 告警系统
知识：笔记应用 → 书签管理器 → 论文库 → 代码片段库
```

每次切换都消耗 1-3 分钟注意力恢复时间。**一天 50 次切换 = 每天浪费 1 小时**。

Worker Agent 的目标：**把所有这些统一到同一个平台**，像操作系统一样自然。

### 1.2 核心设计原则

1. **默认极简**：打开第一眼只有"搜索框 + 最近任务"，像新Mac的桌面
2. **渐进式复杂度**：每层操作只展示当前步骤需要的东西
3. **键盘优先**：80%的功能可以用键盘完成（参考Raycast）
4. **多端统一体验**：同样的操作，在任何设备上获得同等流畅感
5. **远程掌控**：走到哪里都能通过手机控制Agent
6. **本地第一**：所有数据存储在本机，隐私安全完全由用户控制
7. **模块可卸载**：不需要？关掉某个模块即可，不干扰其他

- 复杂操作引导回到Web平台
- 所有远程操作需要**二次确认**（防止误操作）
- 支持设置**免打扰时间**，避免深夜告警
