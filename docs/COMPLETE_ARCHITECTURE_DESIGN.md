│  │  全功能桌面版              移动网页版               轻量入口 │ │
│  │  ·所有模块                 ·核心功能                    ·快捷入口 │ │
│  │  ·快捷键支持                 ·手势操作                   ·消息通知  │ │

### 1.2 为什么选择这种架构

| 考量 | 决策 | 理由 |
|------|------|------|
| 数据存储 | SQLite + IndexedDB | 零依赖，离线可用 |
| 通信协议 | WebSocket + REST | 低延迟+兼容性 |
| 本地AI | Ollama | 开源，支持Qwen2.5/Llama |
| 前端框架 | Vite + React | 最快开发体验 |
| 多端适配 | 统一API + 多壳 | 最小代码复用 |
| Agent框架 | 自定义 | 比LangGraph更轻量 |
| 设计系统 | Penpot | 开源，Figma平替 |

### 2.2 Agent状态机

| 状态 | 含义 | 转换条件 |
|------|------|---------|
| idle | 空闲，等待任务 | 收到任务 → pending |
| pending | 准备中 | 资源就绪 → running |
| running | 运行中 | 完成 → done |
| done | 完成 | 重置 → idle |
| error | 错误 | 重试 → pending/ idle |
| paused | 暂停 | 恢复 → running |
| offline | 离线 | 上线 → idle |

### 2.3 Agent通信

```
┌─────────────┐        ┌──────────────┐        ┌─────────────┐
│  Agent A    │──事件──▶│  Event Bus   │──事件──▶│  Agent B    │
│             │        │  (消息队列)   │        │             │
│  代码Agent  │        ·任务开始     │        │  部署Agent  │
│             │        ·任务完成     │        │             │
└─────────────┘        └──────────────┘        └─────────────┘
                              │
                              ├── 订阅过滤
                              ├── 优先级
                              ├── 重试机制
                              └── 持久化(SQLite)
```
