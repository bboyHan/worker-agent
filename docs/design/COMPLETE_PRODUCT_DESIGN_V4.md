### 1.4 V4.0 相比 v3.0 的核心变化

v3.0 是完整的功能设计+原型，v4.0 增加三个核心能力：

1. **全局浮动Chat面板** — 在任意菜单/页面都能呼出Chat，不用关闭当前页面
2. **任意拖拽分割窗口** — 窗口大小/位置完全自由，像桌面OS一样操控
3. **全局快捷键体系** — 80%操作可通过键盘完成

## 二、核心功能模块设计

### 2.1 全局浮动Chat面板（核心新增）

#### 功能描述
在任意页面（Agent管理/服务器监控/知识库/设置等），通过右下角FAB按钮或快捷键随时呼出Chat面板，对话不受当前页面切换影响。

#### 四级状态设计

| 状态 | 尺寸 | 显示内容 | 触发方式 |
|------|------|---|------|
| 1. 气泡态 | 48×48px | AI图标+未读badge | 自动折叠，点击展开 |
| 2. 半展开 | 280px × 80% | 消息列表+隐藏输入 | 双击/点击FAB |
| 3. 展开态 | 400px × 90% | 消息列表+输入框+工具栏 | 单击FAB/快捷键 |
| 4. 最大化 | 600px × 90% | 输入框扩展+Agent选项面板 | 拖拽边框/双击标题 |

#### 交互设计

```
┌────── 当前页面（任何菜单/功能） ──────────────────┐
│                                                    │
│                                                    │
│              ╔═════════════════╗                   │
│              ║  🤖 Agent Chat  ║ ← 浮动态         │
│              ║  ┌─────────────┐║                   │
│              ║  │ 上次对话记录  │║ ← 消息列表区域    │
│              ║  │             │║                   │
│              ║  └─────────────┘║                   │
│              ║  [消息输入框... ][发送▶]              │
│              ╚═════════════════╝                   │
└───────────────────────────────────────────┘    ▲
                                                  │
                                             FAB按钮
                                             (固定右下角)
```

#### 关键技术点
- 定位：`position: fixed; bottom: 24px; right: 24px;`
- 折叠动画：`transform: translateY(~1000px) scale(0)` → `translateY(0) scale(1)`
- 展开动画：`transform: scale(0.15)` → `scale(1)`
- 跨页面共享：`BroadcastChannel + window.name + localStorage`
- 位置记忆：`sessionStorage` 记住关闭时的位置和状态
- 固定模式：点击📌图标后，面板随页面切换不消失
- 通知Badge：未读消息时FAB显示红点

### 2.2 拖拽分割窗口（核心新增）

#### 功能描述
窗口通过拖拽分割线/拖拽窗口本身来调整大小和位置，支持三种分割模式。

#### 三种分割模式

**模式A：拖拽分割线**
- 拖拽两窗口之间的分割线 → 调整比例
- 实时显示比例（如 42% | 58%）
- 拖拽范围：15%-85%（最小窗口120px）
- 双击分割线 → 恢复50/50

**模式B：拖拽窗口到边缘（Snap Layouts）**
- 拖到屏幕左/右 → 占半屏
- 拖到四角 → 占四分之一
- 类似Win11 Snap Layouts，出现6个布局热力图

**模式C：拖入拆分**
- 将窗口拖到另一窗口中间 → 自动拆分为三面板
- 拖回原窗口 → 自动合并

#### 布局系统

```
布局数据结构：
{
  id: "layout-1",
  name: "默认布局",
  split: "horizontal" | "vertical" | "grid",
  ratio: 0.5,
  primary: { type: "agent", id: "agent-panel" },
  secondary: { type: "layout", children: [...] },
}

示例（三分割）：
{
  split: "horizontal",
  ratio: 0.3,
  primary: { type: "agent", id: "agent-panel" },
  secondary: {
    split: "vertical",
    ratio: 0.5,
    primary: { type: "chat", id: "chat-panel" },
    secondary: { type: "sidebar", id: "sidebar-panel" }
  }
}
```

#### 交互细节
- 拖拽分割线时显示"热力线"（绿色虚线）和实时比例
- 拖到边缘时显示"Snap热力图"（6个布局位置）
- 拖拽时显示面板名称预览
- 布局快照：Ctrl+S保存当前布局，随时恢复
- 键盘微调：箭头键1px/次，Ctrl+Shift+箭头键快速比例

## 五、Top 5 参考项目详细分析

### 5.1 Zed（84k⭐） — 多窗口分割王者

**核心价值**：Zed的窗口分割系统是整个方案的灵感来源。

**关键设计决策**：
- 拖拽分割线时显示"热力图"：绿色+虚线，hover目标区域变蓝
- 布局序列化：Layout.toJSON() 存储/恢复完整布局状态
- 多窗口平铺：类似macOS的Mission Control
- 浮动面板：Terminal/Inspector独立浮动

**借鉴点**：
- 直接参考SplitPane的实现逻辑（拖拽线+比例计算+实时更新）
- 布局快照系统（Zed支持保存多个workspace layout）
- 面板浮动↔嵌入切换

### 5.2 n8n（85k⭐） — 浮动Chat面板直接参考

**核心价值**：n8n的"AI Assist"浮动面板是所有Chat设计参考中最成熟的（有实际大量用户验证）。

**关键设计决策**：
- FAB按钮在任意页面右下角
- 面板拖拽边界碰撞检测（面板不能拖出浏览器）
- 折叠→气泡态动画（scale transform）
- 面板内聊天上下文感知（能感知当前编辑的workflow）

**借鉴点**：
- 直接参考Chat面板的四极状态设计
- 拖拽边界碰撞检测算法
- 跨页面会话共享（BroadcastChannel）

### 5.3 Open WebUI（48k⭐） — Chat折叠/持久化参考

**核心价值**：Open WebUI的Chat面板折叠/展开动画和持久化机制。

**关键设计决策**：
- Chat面板折叠动画（侧滑+缩放）
- localStorage持久化所有对话
- 响应式：移动端自动折叠为底部抽屉

**借鉴点**：
- Chat面板的折叠/展开CSS动画
- localStorage + IndexedDB双写持久化

### 5.4 Raycast（20k⭐开源部分） — 快捷键体系参考

**核心价值**：Raycast的全局快捷键体系是所有效率工具的标杆。

**关键设计决策**：
- Cmd+Space全局触发
- 快捷键面板（显示当前可用快捷键）
- 可自定义快捷键

**借鉴点**：
- 快捷键体系的设计思路（分层+分组+层级导航）
- 快捷键冲突检测

### 5.5 tldraw（46k⭐） — 拖拽引擎参考

**核心价值**：tldraw的任意位置拖拽和碰撞检测引擎。

**关键设计决策**：
- 基于Pointer事件的拖拽引擎
- 拖拽时的吸附/对齐反馈
- 窗口边界碰撞检测
- 拖拽动画（requestAnimationFrame）

**借鉴点**：
- Pointer事件处理（pointerdown/move/up）
- 拖拽边界碰撞检测
- 拖拽时的动画反馈

## 六、推荐技术方案

| 技术 | 方案 | 理由 |
|------|------|------|
| Chat面板 | position:fixed + CSS transition + BroadcastChannel | 成熟、稳定、性能好 |
| 拖拽分割 | Split.js | 成熟库，支持拖拽+边界检测+动画 |
| Snap Layout | 自研 + tldraw碰撞检测API | 需要定制拖拽热力图 |
| 布局存储 | IndexedDB + sessionSnapshot | 持久化+临时快照双写 |
| 快捷键 | mousetrap.js | 轻量、无依赖 |

## 七、设计决策记录

### 决策1：浮动Chat面板状态数量
- 决策：4级状态（气泡→半展开→展开→最大化）
- 理由：2级太少（不够灵活），5级太多（交互复杂）
- 对比：n8n有3级，我们加1级（最大化）

### 决策2：分割窗口采用三种模式
- 决策：拖拽线 + 边缘吸附 + 拖入拆分
- 理由：Zed只用拖拽线，我们增加SnapLayout和拖入拆分，提升灵活性
- 对比：Windows/macOS都有Snap Layout，但窗口拖入拆分是我们创新

### 决策3：快捷键体系分层设计
- 决策：基础快捷键（核心）、Agent快捷键（场景）、自定义快捷键
- 理由：类似VS Code，保证核心功能快捷键稳定，场景快捷键可配置
