# Worker Agent v3.0 — 产品设计文档
## "AI-Native 个人工作平台"

> **一句话定位**：你的 AI 原生个人工作台 —— 一个平台，统一你的办公、开发、知识库、远端指令。
> 类似 Cursor 的极致体验 × Notion 的知识管理 × Slack/飞书 的跨端通讯 × VS Code 的开发能力，全部运行在你自己的 Hermes Agent 系统上。

---

## 1. 产品定位与愿景

### 1.1 重新定义

**不是"Hermes 的 Web UI"**，而是：
- **你的 AI 工作中枢**：所有工作通过 AI agent 完成，这个平台是工作台
- **三支柱模型**：
  - 🖥 **桌面端**（Electron/PWAs）— 全功能开发工作区
  - 📱 **移动端**（小程序/H5/飞书/企微）— 随时随地远控 agent
  - 🧠 **知识库** — ALL-IN 的个人知识体系

### 1.2 核心价值

| 现有方案局限 | Worker Agent v3.0 解法 |
|---|---|
| 各用各的工具（笔记/编辑器/聊天） | 一个平台，AI agent 贯穿一切 |
| 手机只能看不能控 | 飞书/企业微信里直接发指令，agent 干活 |
| 知识库是死的文档 | 可搜索、可引用、可触发 agent 分析的知识图谱 |
| 开发办公割裂 | 代码库 ↔ 知识库 ↔ 任务管理 统一打通 |

## 3. 核心模块设计

### 3.1 🏠 Workspace — 个人工作区

**设计灵感**：Linear 的项目管理 + Notion 的页面组织 + Cursor 的 Agent 驱动

```
Workspace（三个子视图切换）
├── 📋 项目视图
│   ├── 项目卡片（进度、状态、成员）
│   ├── 里程碑时间线
│   ├── 任务分组（Backlog / Doing / Done）
│   └── Agent 参与任务标记
│
├── ✅ 任务视图
│   ├── 看板模式（类似 Linear）
│   ├── 列表模式（筛选 + 排序）
│   ├── 日期视图（日历任务）
│   └── AI 自动任务拆解
│
└── 🔄 Sprint
    ├── 迭代管理
    ├── 进度可视化
    └── Agent 贡献统计
```

**关键交互**：
- 从 Chat 中可以「把对话转为任务」：AI 识别对话中的可执行项，一键创建
- 任务卡片可以 "分配给 Agent"：直接交给某个 Agent 去执行
- 项目看板支持拖拽排序

### 3.2 💬 Agent Chat — 对话面板

**设计灵感**：Cursor Composer 的流畅感 + Slack 的消息格式

```
┌──────────────────────────────────────────────────┐
│ 💬 Agent Chat                                    │
├──────────────────────────────────────────────────┤
│ [Agent Selector ▼] [Session Tab] [⋯ menu]        │
├──────────────────────────────────────────────────┤
│                                                  │
│  ┌────┐                                         │
│  │ 🤖 │ [AI] 好的，我来分析这个需求              │
│  │    │                                          │
│  └────┘  第一步：查看代码库结构                  │
│         第二步：找出相关文件                      │
│         第三步：给出修改建议                      │
│                                                  │
│  ┌────┐                                         │
│  │ 👤 │ 把结果保存到 /docs/ 目录下               │
│  └────┘                                          │
│                                                  │
│  ┌────┐                                         │
│  │ 🤖 │                                      │
│  │    │ ✅ 结果已保存                           │
│  └────┤ /docs/agent-analysis.md                  │
│      │ 文件大小：12KB | 最后修改：2分钟前       │
│      │                                            │
│      │ [查看文件]  [在编辑器中打开]  [知识库引用] │
│                                                  │
│  ────────── 操作区 ──────────                    │
│  [📎附件] [📁文件] [🎤语音] [💡提示词库]         │
│  ┌──────────────────────────────────────┐         │
│  │ 输入消息...                              │        │
│  └──────────────────────────────────────┘         │
│                              [↵] [Cmd+→ 换行]     │
└──────────────────────────────────────────────────┘
```

**核心功能**：
- **Agent 选择器**：支持多 Agent 对话，可随时切换 Agent
- **多轮对话**：完整的会话历史 + 上下文窗口
- **操作卡片**：Agent 的每次操作（如"已创建文件"、"已部署"）都有可交互卡片
- **文件引用**：对话中的文件可以直接预览/跳转
- **知识库引用**：对话中自动关联知识库内容
- **快速操作**：底部快捷工具栏（附件/文件/语音/提示词）
- **快捷键**：`Ctrl+Enter` 发送、`Cmd+K` 插入文件、`Cmd+Shift+V` 粘贴
- **消息类型**：文本、代码块、文件、表格、卡片、任务、Agent 输出

#### 3.2.1 Chat → 任务 转换

这是最重要的生产力转化点：
- 对话中识别可执行项 → 转为 Workspace 任务
- Agent 回复中识别行动项 → 一键创建
- 用户输入"记住这条" → 自动保存为知识库条目


**核心功能**：

  - AI 自动生成大纲/目录
  - 对话式编辑（在文档中直接问 AI）

- **搜索**：全文搜索 + AI 语义搜索
- **版本历史**：文档修改历史，可回滚
- **与 Agent 联动**：
  - 对话中可直接引用知识库
  - AI 基于知识库回答
  - Agent 可读写知识库

#### 3.5.2 飞书/企微机器人接入

**场景**：你在路上，用飞书给 Agent 发消息，Agent 直接执行任务

```
用户侧（手机飞书）                  Worker Agent 侧
──────────────────────              ────────────────────
┌──────────────────┐               ┌──────────────────┐
│ 飞书聊天窗口       │               │ Hermes Gateway    │
│                  │  消息         │   (Webhook)       │
│ [你]：帮我查一下  │ ──────────→ │                   │
│   服务器状态      │               │ → 调度 agent      │
└──────────────────┘               │ → 执行任务         │
                                    │ → 结果回传        │
                                    │ ←────────────── │
                                    │ ← 结果推送        │
                                    │                   │
用户侧（手机飞书） ←── 推送响应      │                   │
┌──────────────────┐               ┌──────────────────┐
│                  │               │                  │
│ [Agent]：服务器   │               │                  │
│ 状态正常，CPU 42% │ ←────────── │                  │
│ 内存 64%         │               │ [结果卡片]       │
└──────────────────┘               └──────────────────┘
```

**功能**：
- 📨 **消息直连**：在飞书/企微里直接向 agent 发送指令
- 📊 **状态推送**：Agent 执行结果实时推送飞书/企微
- ⚠️ **告警推送**：任务失败、Agent 离线等告警推送
- 🎤 **语音指令**：飞书语音转文字 → Agent 执行
- 📎 **附件处理**：飞书附件 → Agent 分析
- 🔗 **快捷卡片**：飞书消息卡片，直接操作任务/查看状态

## 6. 交互设计原则

### 6.1 核心交互原则

| 原则 | 说明 | 参考 |
|------|------|------|
| **一切皆可搜索** | `Ctrl+K` 是万能入口 | VS Code |
| **零延迟反馈** | 所有操作立即反馈 | Linear |
| **Agent 优先** | 对话是主要交互方式 | Cursor |
| **上下文感知** | AI 始终了解你的工作 | Copilot |
| **渐进式披露** | 复杂功能逐步展现 | Vercel |
| **统一的多端体验** | 手机桌面功能一致 | Linear |

### 6.2 动效设计
- **微动效**：页面切换、卡片点击、加载状态
- **骨架屏**：加载时使用骨架屏过渡
- **流式输出**：AI 回复采用打字机效果
- **折叠面板**：可折叠面板减少干扰

## 7. 与 CLI 的关系

| 操作 | 最佳方式 |
|------|----------|
| 日常对话 | 🖥 GUI（Chat） |
| 新建 Agent | 🖥 GUI 或 CLI（看你偏好） |
| 查看日志 | 🖥 GUI（可视化更好） |
| 配置 API Key | 🖥 GUI（更直观） |
| 服务器调试 | 🖥 SSH/CLI |
| 快速创建会话 | 📱 手机/飞书（最快） |

**设计哲学**：GUI 和 CLI 是互补的，GUI 管"管理/观察"，CLI 管"执行"。

## 8. API 设计

### 8.1 REST API（对外暴露）

```yaml
# 统一 API 端点
POST   /api/v1/agents/chat        # Agent 对话
GET    /api/v1/sessions             # 会话列表
POST   /api/v1/sessions             # 创建会话
GET    /api/v1/sessions/:id         # 会话详情
GET    /api/v1/files/tree           # 文件树
GET    /api/v1/files/:id/preview    # 文件预览
POST   /api/v1/tasks                # 创建任务
GET    /api/v1/tasks                # 任务列表
GET    /api/v1/knowledge/search     # 知识库搜索
POST   /api/v1/knowledge/import     # 导入知识库
GET    /api/v1/flows                # 工作流列表
POST   /api/v1/flows/trigger        # 触发工作流
GET    /api/v1/monitor/metrics      # 监控数据
POST   /api/v1/remote/notify        # 远程推送通知
```

### 8.2 Webhook 接入（飞书/企微）

```yaml
# 飞书 Webhook 接入
POST /api/v1/webhooks/feishu       # 接收飞书消息
GET  /api/v1/webhooks/feishu/info  # Webhook 配置信息

# 企业微信 Webhook 接入
POST /api/v1/webhooks/wecom        # 接收企业微信消息
GET  /api/v1/webhooks/wecom/info   # Webhook 配置信息

# API Key 配置
POST /api/v1/webhooks/feishu/config
GET  /api/v1/webhooks/feishu/config
# 返回：app_id, app_secret, agent_id 供飞书机器人配置
```
