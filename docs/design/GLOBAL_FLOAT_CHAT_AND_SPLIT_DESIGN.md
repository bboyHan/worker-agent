# Worker Agent v4.0 — 浮动聊天 + 拖拽分割设计

## 一、Top 5 优秀开源项目调研

### 筛选标准
- 开源免费优先
- 有实际用户量（>5k stars 或广泛使用）
- 设计可借鉴性（浮动聊天、分割窗口、全局交互）
- 技术栈友好（Web前端可复用）

---

### 1⃣  Zed (zed-industries/zed) — 84k⭐

**类型**：高性能代码编辑器 + 多窗口系统

**核心亮点**：
- **多窗口平铺**：类似macOS的Mission Control，窗口可平铺分割
- **浮动面板系统**：Terminal、Inspector、Find-All都可浮动成窗
- **Split布局引擎**：水平/垂直/网格分割，拖拽热区切换布局
- **多光标编辑**：类似Sublime的多selection

**可借鉴之处**：
- 窗口分割方式（拖拽分割线→出现布局热力图→点击选格局）
- 浮动面板的生命周期管理（浮动↔嵌入切换）
- 窗口状态序列化（保存/恢复布局）

**不足**：
- 纯编辑器，无Chat设计参考
- Rust生态，直接代码复用有限

**推荐度**：⭐⭐⭐⭐⭐（窗口系统核心参考）

---

### 2⃣  n8n (n8n.io) — 85k⭐

**类型**：开源AI Agent工作流平台

**核心亮点**：
- **浮动AI聊天面板**：点击页面右下角"AI Assist"按钮，呼出浮动聊天
- **上下文感知**：聊天面板能感知当前编辑的workflow context
- **可折叠/可固定**：面板可最小化到桌面边，可固定在侧边
- **工作流内直接调用**：在workflow编辑面板中直接对话Agent
- **面板拖拽**：浮动面板可拖到任意位置

**可借鉴之处**：
- 浮动Chat的触发方式（圆形FAB右下角 + 快捷键Alt+Shift+K）
- 面板在不同页面下的状态保持（搜索页/Agent页/设置页都保持对话上下文）
- 面板最小化→气泡→半展开→全展开四级状态

**推荐度**：⭐⭐⭐⭐⭐（浮动Chat直接参考）

---

### 3⃣  Raycast (raycast.com/opensource) — 开源部分 20k+⭐

**类型**：macOS效率工具 + 弹窗面板系统

**核心亮点**：
- **全局弹出面板**：Cmd+Space呼出，在所有应用之上显示
- **面板内搜索**：实时搜索 + 键盘导航 + 预览
- **面板内操作**：不用关闭面板就能执行命令
- **多面板模式**：Search / Shortcuts / Windows / Extensions

**可借鉴之处**：
- 跨页面面板状态保持
- 面板大小/位置记忆
- 快捷键呼出/收起模式

**不足**：
- 商业产品，部分功能不开源

**推荐度**：⭐⭐⭐⭐（全局面板设计参考）

---

### 4⃣  Open WebUI / Ollama Web (Open WebUI) — 48k⭐

**类型**：开源ChatGPT替代Web UI

**核心亮点**：
- **浮动聊天面板**：页面右侧浮动聊天窗口，可折叠/展开
- **多会话管理**：左侧栏管理多个Chat会话
- **对话历史持久化**：切换页面后对话不丢失
- **响应式布局**：移动端自动折叠为底部抽屉

**可借鉴之处**：
- Chat面板折叠/展开动画（侧滑+缩放）
- 对话持久化策略（localStorage + IndexedDB双写）
- 多Agent对话切换

**推荐度**：⭐⭐⭐⭐⭐（Chat面板直接参考）

---

### 5⃣  tldraw (tldraw.com) — 47k⭐

**类型**：开源白板SDK（可拖动/缩放无限画布）

**核心亮点**：
- **任意位置元素拖动**：Canvas上任何元素都可以拖到任意位置
- **拖拽热区**：元素悬浮在拖拽区域时出现操作提示
- **拖拽吸附**：拖拽时智能吸附对齐线
- **多元素分组**：可框选+移动一组元素

**可借鉴之处**：
- 拖拽引擎（Pointer事件 + transform + requestAnimationFrame）
- 拖拽时的吸附/对齐反馈
- 窗口边界外的碰撞检测

**不足**：
- 白板工具，非Chat面板

**推荐度**：⭐⭐⭐⭐（拖拽技术参考）

## 二、全局浮动Chat面板设计

### 2.1 需求分析

用户在任意菜单/页面下，都需要随时与Agent对话：
- 管理Agent时，想直接问Agent"帮我调整模型"
- 查看服务器时，想问Agent"帮我分析CPU瓶颈"
- 搜索知识库时，想问Agent"帮我总结这个知识点"
- **不能在当前页面关闭后丢失对话**

### 2.2 设计方案

```
┌─────────────────────────────────────────────────────────┐
│                        当前页面内容                        │
│                                                         │
│   （任何菜单/页面/功能）                                │
│                                                         │
                    ╔═══════════════════╗
                    ║  ┌───┐  Worker  ║
         最小化态   ║  │🤖│  Agent  > ║  半展开态
         (气泡)    ║  └───┘  [Chat]   ║  (400px宽)
                    ║                  ║
                    ║  ┌────────────┐  ║
                    ║  │ 输入消息... │  ║  展开态
                    ║  └────────────┘  ║  (600px宽)
                    ║  [发送▶]         ║
                    ╚═══════════════════╝
                           ↑
                    右下角FAB按钮呼出
                       或 ⌘+Shift+K
```

### 2.3 四级状态

| 状态 | 宽度 | 高度 | 功能 | 触发方式 |
|------|------|------|------|---------|
| 1. 气泡态 | 48px | 48px | 显示通知badge，点击展开 | 自动折叠 |
| 2. 半展开 | 280px | 80% | 消息列表，输入框隐藏 | 双击/点按钮 |
| 3. 展开态 | 400px | 90% | 消息列表+输入框+工具栏 | 单击/快捷键 |
| 4. 最大化 | 600px | 90% | 输入框扩展，显示Agent选项 | 拖拽边框 |

### 2.4 核心交互

- **呼出**：点击右下角FAB / 按 ⌘+Shift+K / 呼出后面板在右下角
- **关闭**：点击面板外 / 再次按快捷键 / 折叠到气泡
- **拖拽**：面板标题栏可拖拽（但限制在Viewport内）
- **最小化**：拖拽边缘收缩为气泡 / 点 × 折叠
- **固定**：点 📌 图标固定面板，不随页面切换消失
- **通知**：未读消息时FAB出现红点badge
- **恢复位置**：关闭后面板自动恢复到上次位置

### 2.5 技术实现

- **定位**：`position: fixed; bottom: 24px; right: 24px;`
- **动画**：CSS transition `transform + opacity + width + height`
- **折叠态**：`transform: scale(0.15)` 到 `transform: scale(1)`
- **消息持久化**：IndexedDB + localStorage双写
- **跨页面共享**：BroadcastChannel + window.name + sharedWorker
- **拖拽引擎**：Pointer事件 + 边界碰撞检测（参考tldraw拖拽引擎）

## 三、任意拖拽分割窗口设计

### 3.1 需求分析

用户希望：
- 窗口之间可以通过拖拽分割线调整比例
- 任意位置拖拽可以改变窗口布局（如A在左B在右 → 拖到B在左A在右）
- 窗口可以浮动成独立区域
- 拖拽时有视觉反馈（热力图/热力线）

### 3.2 设计方案

```
┌─────────────────────────────────────────────────────┐
│  窗口A [280px]  |  拖拽线  |  窗口B [320px]       │
│   ══════════════|═════════|════════════════════    │
│     Agent管理     |   ●    |     Chat面板           │
│                   |   ←→   |     ┌─────────────┐   │
│                   |  拖动   |     │ 消息列表    │   │
│                   │   ●   │     └─────────────┘   │
├─────────────────────────────────────────────────────┤
│                      Chat面板(浮动)                  │
│           ┌─────────────────────────────────┐       │
│           │  🤖 AI Assist (可拖拽浮动)      │       │
│           │  [输入消息...][发送]             │       │
│           └─────────────────────────────────┘       │
└─────────────────────────────────────────────────────┘
```

### 3.3 三种分割模式

**模式1：拖拽分割线（SplitPane）**
- 拖拽两个窗口之间的分割线 → 调整宽度比例
- 拖拽上限/下限：最小子窗口 120px
- 拖拽时实时显示宽度比例（如 40:60）

**模式2：拖拽窗口到边缘（Snap Layout）**
- 将窗口拖到屏幕左/右/上边缘 → 吸附占满半边/四分之一
- 类似Windows Snap Layouts，出现6个布局热力图
- 释放时自动填充对应区域

**模式3：拖拽窗口到分割区域**
- 窗口A拖到窗口B旁边 → 窗口C自动插入，形成三分割
- 拖拽线出现时显示 "+ 新分割" 热力图
- 拖回原窗口 → 自动合并

### 3.4 布局系统

```
┌─ Layout System ─────────────────────────────────────┐
│                                                      │
│  布局存储: localStorage(当前布局) + IndexedDB(历史布局)│
│                                                      │
│  核心结构:                                           │
│  Layout = {                                       │
│    split: "horizontal" | "vertical",              │
│    primary: Window | Layout,                       │
│    secondary: Window | Layout,                     │
│    ratio: 0.5 (between 0.15 and 0.85)             │
│  }                                                 │
│                                                      │
│  示例（三分割）:                                     │
│    split: "horizontal", ratio: 0.3                 │
│    primary: {                                        │
│      split: "vertical", ratio: 0.5                 │
│      primary: "agent-panel",                        │
│      secondary: "chat-panel",                        │
│    }                                              │
│    secondary: "sidebar-panel",                      │
│  }                                                 │
└──────────────────────────────────────────────────────┘
```

### 3.5 交互细节

- **拖拽反馈**：拖拽时显示"热力线"（绿色+虚线），hover在目标区域变蓝
- **比例显示**：拖拽分割线时显示实时比例（如 `▮ 42% | 58% ▮`）
- **重置按钮**：拖拽线中间点 → 恢复50/50
- **双击**：双击分割线 → 自动调整到"合适"比例
- **键盘**：Arrow keys微调（1px/次），Ctrl+Shift+Arrow键快速比例

## 四、其他价值功能推荐

基于对Top 5项目的分析，增加以下功能值得推荐：

### 4.1 布局快照/恢复（来自Zed）
- **功能**：保存当前窗口布局到快照，随时恢复
- **价值**：用户可按不同场景保存布局（开发布局 / 管理布局 / 分析布局）
- **实现**：`Layout.toJSON()` + localStorage存储多个snapshot

### 4.2 全局搜索（来自Raycast）
- **功能**：Cmd+Space全局搜索所有功能/Agent/文件
- **价值**：不离开当前页面也能快速导航到任何功能
- **实现**：输入框 + FTS5搜索 + 键盘导航 + 实时预览

### 4.3 键盘快捷键体系（来自所有Top项目）
- **功能**：80%操作可通过快捷键完成
- **价值**：效率翻倍，提升用户黏性
- **建议**：参照VS Code的快捷键体系，同时支持Cmd和Ctrl

### 4.4 面板内上下文感知（来自n8n）
- **功能**：Chat面板根据当前页面自动调整Agent角色
- **价值**：用户不用手动切换Agent，对话更精准
- **示例**：在服务器页面 → Chat自动切换到"运维Agent"角色

### 4.5 响应式设计（来自Open WebUI）
- **功能**：窗口/面板在不同屏幕尺寸自动调整
- **价值**：H5移动端也能完整使用
- **实现**：CSS媒体查询 + 面板折叠/展开策略

## 五、设计决策

### 推荐技术方案

| 模块 | 技术 | 参考 |
|------|------|------|
| 浮动Chat面板 | Position fixed + CSS transition + BroadcastChannel | n8n + Open WebUI |
| 拖拽分割 | Split.js 或 自研Pointer拖拽引擎 | Zed + tldraw |
| 布局存储 | Layout JSON序列化 + IndexedDB | Zed |
| Chat持久化 | IndexedDB + localStorage fallback | Open WebUI |
| 全局搜索 | FTS5 + Keyboard navigation | Raycast |

### 优先级

1. ⭐⭐⭐⭐⭐ 浮动Chat面板（核心差异功能）
2. ⭐⭐⭐⭐⭐ 拖拽分割窗口（窗口系统基础）
3. ⭐⭐⭐⭐ 布局快照（效率功能）
4. ⭐⭐⭐⭐ 全局搜索（导航效率）
5. ⭐⭐⭐ 上下文感知Chat（智能化增强）
