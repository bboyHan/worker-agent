# Worker Agent v2.0 — 产品设计文档

> 一句话定位：**把你的 Hermes Chat 搬到平台里可视化管控**

## 1. Product Vision（产品愿景）

### 1.1 核心问题
- 当前 Hermes Agent 完全在 CLI 里操作，命令行不够直观
- 多 Agent + 多会话场景下，无法一目了然地看到所有 Agent 状态
- 会话切换、搜索、管理都依赖于终端命令，不够方便
- 无法图形化配置 API Key、模型选择等设置

### 1.2 产品定位
Worker Agent 是一个 **Hermes Agent 的可视化管控平台**：
- ✅ 在浏览器里管理你的多 Agent
- ✅ 在浏览器里管理多会话（新建/切换/搜索/历史）
- ✅ 在浏览器里配置 Hermes 设置（模型、API Key 等）
- ✅ 在浏览器里直接和 Agent 对话（聊天界面交互）
- ❌ 不是 CLI 替代品（CLI 依然有用）
- ❌ 不是新的 AI Agent 框架（它是 Hermes 的前端）

### 1.3 目标用户
你一个人用，管理自己平台里的 Hermes 系统。

## 5. 设计原则

### 5.1 设计定位
- **管理型 GUI**，不是 CLI 替代品
- **可视化**：状态一目了然，避免命令式操作
- **Hermes 原生**：管理的是你现有的 Hermes 系统，不是新建框架

### 5.2 交互原则
- 所有模块都能通过左侧图标栏快速跳转
- 会话 ↔ Agent 是一等公民（核心模型）
- 右侧面板是持久性的（不会丢失上下文）
- 全局搜索（Ctrl+P）可跨模块搜索

### 5.3 不在 MVP 范围内的
- 实时协作（单用户场景）
- WebSocket 实时推送（用轮询即可）
- 移动端 APP（Web 响应式即可）
- Agent Market（插件市场可以手动安装）

## 6. 与 CLI 的关系

| 场景 | 用什么 |
|------|--------|
| 日常对话、快速创建 | 🖥 CLI（更快） |
| 查看多 Agent 状态 | 🌐 Web GUI（更直观） |
| 会话管理、搜索 | 🌐 Web GUI |
| 配置 API Key、模型 | 🌐 Web GUI |
| 调试、查看日志 | 🖥 CLI（更精确） |
| 服务器运维 | 🖥 CLI / SSH |

**定位**：GUI 管"管理视角"，CLI 管"执行视角"，两者互补。
