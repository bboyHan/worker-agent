# Feature 13 — 容器监控设计

> 日期: 2026-06-02
> 优先级: 低
> 状态: 完整设计完成

## 1. 模块定位

监控运行中容器的资源使用情况（CPU/内存/网络/磁盘），在终端内以图表形式展示。

## 2. 功能边界

### 2.1 包含的功能

| 功能 | 描述 |
|------|------|
| 实时监控 | CPU/内存/网络/磁盘使用率 |
| 历史图表 | 时间序列折线图（终端绘制） |
| 阈值告警 | 超阈值时触发 Feature 11 告警 |
| 容器对比 | 多容器资源对比 |
| 资源限制 | 设置容器资源上限/下限 |

### 2.2 排除的功能

- 不涉及 Prometheus/Grafana 等重型监控
- 不提供系统级监控（NodeExporter 等，仅容器维度）

## 3. 数据模型

数据存储在 `data/container-monitor/` 目录：

```json
{
  "monitors": [
    {
      "id": "mon_001",
      "containerId": "abc123",
      "containerName": "my-api",
      "interval": "10s",
      "historyDays": 7,
      "thresholds": {
        "cpu_percent": 80,
        "memory_mb": 512
      }
    }
  ],
  "history": {
    "abc123": [
      {"timestamp": "2026-06-02T12:00:00", "cpu": 45.2, "mem_mb": 256, "net_in": 1024, "net_out": 2048},
      {"timestamp": "2026-06-02T12:00:10", "cpu": 52.1, "mem_mb": 268, "net_in": 1100, "net_out": 2200}
    ]
  }
}
```

## 5. 实现方案

- 使用 `docker stats --no-stream` 定时采集数据
- CPU/内存使用率通过 ASCII 柱状图绘制（使用 ink- 组件/ansi-art）
- 历史数据存储于 JSON 文件 `data/container-monitor/history.json`
- 历史图表使用 Unicode 字符绘制折线图
