### 四、数据库必要性分析

| 场景 | 数据量 | 纯文件可行？ | SQLite必要性 | 结论 |
|------|------|------|------|------|
| 任务看板 | 100-500条 | ✅ JSON | ❌ 无需 |
| 知识库 | <1000篇 | ✅ MD文件 | ❌ 无需 |
| 全局搜索索引 | <10MB | ✅ JS对象 | ❌ 无需 |
| 配置 | 1文件 | ✅ JSON | ❌ 无需 |
| SSH密钥 | <20条 | ❌ 明文不安全 | ✅ 加密存储好 |
| IM凭证 | <10条 | ⚠️ 加密需处理 | ✅ 更安全 |
| 服务器配置 | 5-50台 | ✅ JSON | ❌ 无需 |
| 告警历史 | 持续增长 | ⚠️ 追加文件 | ✅ 查询更方便 |
| 任务日志 | 持续增长 | ⚠️ 追加文件 | ✅ 查询/筛选好 |
| Agent状态 | 12个 | ✅ JSON | ❌ 无需 |

### SQLite必要性总结

- SQLite的优势：百万级数据、复杂JOIN
- 本平台数据总量 < 3MB，单表<1000 | 无需SQLite
- 全量数据<3MB单表<100条，纯文件完全够用
- **结论：不需要单独安装数据库。** 使用纯文件方案即可。
